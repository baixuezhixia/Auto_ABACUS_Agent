# Agent Rebuild I/O Spec (Simple)

This guide defines the minimum required input and the standard output locations.

## 1) Minimal Input You Need To Prepare

### 1.1 Config file

Use `config.yaml` (copy from `config.example.yaml`) and fill:

- `abacus.executable`
- `abacus.run_mode`
- `abacus.np`
- `abacus.pseudo_dir`
- `abacus.orb_dir`

### 1.2 Structure file

Provide a valid ABACUS `STRU` file path to `--structure`.

For plane-wave (`basis_type = pw`) calculations, the required sections are usually:

- `ATOMIC_SPECIES`
- `LATTICE_CONSTANT`
- `LATTICE_VECTORS` (or use `latname` in INPUT)
- `ATOMIC_POSITIONS`

## 2) Standard Runtime Input Generated By Agent

For each task folder, agent writes:

- `INPUT`
- `STRU`
- `KPT`

`INPUT` is standardized for ABACUS v3.9+

- Starts with `INPUT_PARAMETERS`
- Uses `orbital_dir` (not `orb_dir`)
- Uses `stru_file = STRU` and `kpoint_file = KPT`

## 3) Standard Output You Will Read

Given `--work-dir ./runs`, the outputs are:

- `./runs/<task_folder>/INPUT`
- `./runs/<task_folder>/STRU`
- `./runs/<task_folder>/KPT`
- `./runs/<task_folder>/OUT.<suffix>/running_*.log`
- `./runs/report.json`

`report.json` is the main machine-readable result.

Key fields:

- `tasks[]`: task plan
- `execution[]`: per-task run result
- `execution[].status`: success/failed
- `execution[].metrics`: parsed values (energy, convergence flags, etc.)

## 4) Where To Check Failures Quickly

If a run fails, check in this order:

1. `running_*.log` in `OUT.<suffix>/`
2. `warning.log` in `OUT.<suffix>/`
3. `report.json` for `stderr_tail` and `metrics`
